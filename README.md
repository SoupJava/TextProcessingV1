# TextProcessingV1
Vue3-SpringBoot-FastAPI框架版本
# 1.1 开发背景
随着我国教育水平的不断提高，每年的高校毕业生也越来越多。当高校毕业生在撰写毕业论
文时，因为部分学生的语文功底不好或者国语水平不好，会经常出现毕业论文中有大量的如错
字，错句，措辞不当等低级错误。同时面对篇幅很长的毕业论文来说，在修改论文时往往会修改
很多版本，而当版本过多时，对同一篇论文的版本管理也是一件令高校毕业生十分头疼的一件事
情。

# 4.创新点 
4.1 论文版本管理
我们通过调查发现，在论文修改的过程中，当缺少一个良好的版本管理时，改版次数过
多后，查找论文便是一个非常麻烦的事情，而目前市场中缺少专门的论文版本管理的在线平
台。

因此该平台的在论文上传时会跟上一版进行一个相似度比较，默认当相似度在 98%-100%
时，平台会弹出提醒，提醒用户相似度太高；当相似度在 80%-98%时，上传后的文档自动添
加一个小版本更新；当相似度在 50%-80%时，上传后的文档自动添加一个大版本更新；当相
似度在 0%-50%时，平台会提醒用户相似度太低是否是上传了错误的论文。相似度对比值用
户也可以自己修改。

通过该论文版本的管理，用户可以直接将改版后的文档进行上传，方便用户的保存备份
和使用时候的查找，有效提高效率。


4.2 文本纠错功能
当用户在进行论文创作时，很容易出现错字，错标点，语句不通顺等情况，而这些错误
自己往往是无法察觉的。

市场中的其他辅助平台在处理文本纠错时，大多数都是通过上传语句或者文档来直接返
回纠错后的文档，但是当面对长篇的论文时，直接返回结果往往不能让用户知道到底修改了
什么地方，因此该平台在设计文本纠错功能时，将该功能分成了两个步骤：

1. 用户上传自己的原文档，后台处理后会自动反正错误报告。其中错误报告中包含原文档
内容和错误的语句，以及该语句的坐标和推荐的正确语句，方便用户了解修改的地方。

2. 当用户拿到错误报告后，可以直接查看错误报告，当发现推荐的正确语句并不符合自己
的心理预期时，可以直接在该处进行修改，当确保修改后的错误语句都符合心理预期后，
可以直接将该错误报告上传。平台会自动将错误的语句进行替换并返回。
